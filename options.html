Github limits requests to 60 per hour for unauthenticated requests. The extension caches all requests to prevent unnecessary API calls. The cache expires in one week which means you won't be able to see latest information about neither the user nor the repository. Although it seems long, it should be fine for most cases. If you want to go beyond the limit (5000 requests/hour) then you will need to create a token from https://github.com/settings/applications and use the token below. In this case, cache won't be effective.
